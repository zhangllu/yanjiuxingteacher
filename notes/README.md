# 项目笔记 (Notes)

这个目录用于存放项目相关的笔记、文档和学习资料。

## 目录结构建议

```
notes/
├── meetings/          # 会议记录
├── research/          # 调研资料
├── learning/          # 学习笔记
├── decisions/         # 决策记录
└── references/        # 参考资料
```

## 使用建议

### 会议记录
记录项目会议的关键讨论和决策：
- 会议时间、参与人员
- 讨论议题
- 决策结果
- 行动项

### 调研资料
存放市场调研、用户调研、竞品分析等资料：
- 用户访谈记录
- 问卷调查结果
- 竞品分析报告
- 行业研究报告

### 学习笔记
记录技术学习和最佳实践：
- 技术文档阅读笔记
- 教程学习笔记
- 问题解决方案
- 最佳实践总结

### 决策记录
使用ADR (Architecture Decision Records) 格式记录重要决策：
- 背景：为什么需要做这个决策
- 决策：我们决定做什么
- 后果：这个决策带来的影响

### 参考资料
存放项目相关的参考文档：
- API文档
- 设计规范
- 编码规范
- 第三方服务文档

## 文档格式

建议使用Markdown格式，便于版本控制和协作。

## 注意事项

1. **及时记录**：重要信息要及时记录，避免遗忘
2. **结构清晰**：使用清晰的目录结构和文件命名
3. **定期整理**：定期整理和归档笔记
4. **保护隐私**：不要在笔记中记录敏感信息（密码、密钥等）
