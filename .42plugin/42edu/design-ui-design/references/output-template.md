# UI 规格输出模板

生成 `spec/design/ui.spec.md` 时使用以下结构：

```markdown
# [项目名称] UI 设计规格

<meta>
  <document-id>[project]-ui-spec</document-id>
  <version>1.0.0</version>
  <project>[项目名称]</project>
  <type>UI 设计规格</type>
  <created>[日期]</created>
  <tech-stack>Next.js 15+, React 19+, Tailwind CSS v4, shadcn/ui, Zustand</tech-stack>
</meta>

## 1. 智能分析结论

### 1.1 应用类型
**结论**: [SPA / MPA / 混合]
**理由**: ...

### 1.2 导航结构
**类型**: [顶部 / 侧边 / 底部 / 混合]
**主导航**: ...

### 1.3 配色方案
**主色相**: [N]° [颜色名]
**OKLCH 配置**: [CSS 变量定义]

## 2. 设计系统
[设计令牌、字体配置]

## 3. 页面布局
[响应式断点、布局结构]

## 4. 组件规格
[shadcn/ui 组件列表、自定义组件]

## 5. 状态管理
[Store 定义、持久化配置]

## 6. 功能独立原则
[三条规则、Mock 模式配置]

## 7. Mock 数据
[Mock 数据结构、AI 响应生成器]

## 8. 核心功能实现
[P0/P1 功能列表、实现规范]

## 9. 交互模式
[加载状态、反馈模式、空状态]

## 10. 无障碍性
[WCAG 检查清单]

## 11. 扩展点
[数据库迁移路径、API 实现路径、认证集成路径]

## 12. 验收检查清单
[完整检查项]
```

## 关键章节说明

| 章节 | 必须性 | 说明 |
|------|--------|------|
| 1. 智能分析 | 必须 | 核心设计决策 |
| 6. 功能独立 | 必须 | 确保可测试性 |
| 7. Mock 数据 | 必须 | 确保即时可用 |
| 12. 验收检查 | 必须 | 质量保证 |

> **说明**：项目初始化由技能执行时自动完成，不包含在规格文档中。
